公司内部多个业务系统在关键事件发生时，需要调用外部合作方提供的 HTTP(S) API 进行通知。比如：

* 当用户通过第三方广告系统引流并成功注册后，我们需要将该信息告知相应的广告系统。
* 当用户订阅付款成功后，我们要将用户付款成功的信息通知相应的 CRM 系统。
* 当用户购买商品后，库存会发生变动，我们要通知库存系统做库存变化通知。
 ……

各合作方的 API 地址和请求格式（包含 Header、Body 等）均由合作方定义，不同业务系统需要根据对接方的要求传递不同的内容。需要注意的是，这些 API 调用的返回值业务系统无需感知，只需确保请求被可靠地发出。

为保证调用的可靠性，并降低各业务系统的实现和维护复杂度，需要设计一个“API 通知系统”。该系统对内通过标准化的 HTTP API 提供服务，业务系统只需调用该 API，传入目标地址和请求内容；该系统需确保该 API 通知能够稳定、可靠地送达外部系统。
